YTPing-网络质量监控 · 使用说明
返回监控

本文介绍 YTPing 监控平台的主要功能与常见操作。默认访问地址：http://<服务器>:3000。

登录与安全

  • 首次部署后使用默认管理员密码登录（请尽快在右上角菜单中修改密码）。
  • 登录状态保存在浏览器本地；退出登录或改密后需重新登录。

监控目标

添加 / 编辑

  • 点击添加目标，填写名称、地址（域名或 IP）、探测间隔（500ms～60s，默认 1 秒）。
  • 可设置分组、标签（多个标签用英文逗号分隔），便于首页筛选。
  • 卡片上显示 ID n 为系统内部编号，定时任务「指定 ID」范围时使用。

卡片菜单

  • 编辑、克隆（复制配置并自动命名）、启用/停用、删除。
  • 点击卡片进入详情：实时、延迟趋势、丢包记录、历史统计。

首页看板

  • 顶部统计：正常 / 告警 / 异常 / 停用数量（基于近 1 分钟探测）。
  • 可按分组、标签、关键词筛选；支持刷新自动更新。
  • 卡片内20 格色块表示最近约 20 秒每秒探测结果（绿/黄/红/灰）。
  • 均值延迟、丢包率标注为「近 1 分钟」。

详情页说明

  • 实时：近 5 分钟折线图，SSE 推送；显示当前样本与丢包率。
  • 延迟趋势：自定义 1H～7D 或时间范围，查看 Min/Avg/Max 与丢包率曲线及区间统计。
  • 丢包记录：分页浏览历史丢包时间点。
  • 历史统计：7 天前数据压缩为延迟分布桶与丢包比例；可筛选时间并手动压缩更早的原始数据。

批量启停

  • 导航栏 批量启停：对「当前筛选结果 / 全部 / 指定分组 / 指定标签」一次性启用或停用。
  • 停用后不再 ping，适合夜间关机、维护窗口，避免无意义丢包入库。

定时启停

  • 按容器本地时间每天在设定时刻执行（建议配置环境变量 TZ=Asia/Shanghai）。
  • 范围：全部目标、分组/标签（可多选）、或指定 ID（逗号分隔，见卡片 ID）。
  • 典型用法：22:00 停用 + 08:00 启用 两条规则。

部署提示

  • 镜像需 --cap-add=NET_RAW 以支持 ping；数据目录挂载到 /data（如 -v /path/ytping:/data）。
  • 端口映射示例：-p 3000:3000。离线包导入与运行参数见项目 DOCKER-YTPING.md。
  • 仅暴露 Web 端口；数据库为容器内 SQLite 文件，无单独数据库端口。

界面主题

  • 右上角可切换深色 / 浅色主题，偏好会保存在浏览器中。
